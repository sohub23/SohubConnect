bound Route
What is an Inbound Route?
An Inbound Route defines how incoming calls are handled when someone dials one of your numbers. It connects an incoming Caller ID (DID) to a specific Call Flow, such as ringing extensions, playing an IVR, or forwarding calls.
Inbound Routes are essential to receive calls correctly.

Create an Inbound Route
Go to Voice → PBX → Inbound Route
Click Add Inbound Route
Fill in the following fields:
Inbound Route Name (Required)
Enter a descriptive name for the route
Example: Main Office, Support Line, Sales Number
Select Caller ID (Required)
Choose the phone number that will receive incoming calls
This is usually a purchased or approved DID
Select Call Flow (Required)
Choose how the incoming call should be handled
Example: IVR, Ring Group, Extension, Time-based flow
Route Status (Required)
Active: Route will receive calls
Inactive: Calls will not be received
Click Save
Once saved, the inbound number becomes live immediately if the route is active.

Inbound Route List
The Inbound Route list shows all configured inbound routes with the following details:
Created Time – Date the route was created
Inbound Name – Route name
Caller ID – Incoming phone number
Call Flow – Assigned call handling logic
Assigned To – Customer or company
Status – Active or Inactive
Action – Edit or Delete the route

Common Use Cases
Route a main business number to an IVR
Send support calls to a Ring Group
Forward calls to different flows based on time
Assign different numbers to different departments

Important Notes
An inbound route must have a Call Flow to work
If the route is inactive, calls will fail or disconnect
Caller ID must be approved and active

Troubleshooting
If incoming calls are not working:
Check that the route status is Active
Confirm the correct Caller ID is selected
Ensure the assigned Call Flow is published and active
Verify the number is not used in another inbound route

Ring Groups allow you to route incoming calls to multiple extensions at the same time or in a defined order. This ensures that calls are answered quickly by available team members.

Click Add Ring Group.
Fill in the required fields:

Approved Trunks
Overview
Approved Trunks show the list of SIP / PSTN trunks that are requested by the customer and approved by Sohub or the telecom provider. Trunks are required to make outbound and inbound calls from Sohub Connect.
Each trunk goes through an approval process. Until approval, the trunk remains inactive and cannot be used for calling.

Accessing Approved Trunks
Navigate to: Voice → Approved Trunks
This page displays all trunk requests along with their current status.

Column Description
Requested Time: Date when the trunk was requested by the customer.
Approved Time: Date when the trunk was approved by Sohub / provider. Shows Not Approved Yet if pending.
Merchant No: The customer or company name under which the trunk is assigned.
Provider: Telecom or SIP provider name.
Trunk Number: The main trunk DID or number assigned by the provider.
Trunk Username: SIP username (if applicable).
Trunk Password: SIP password (hidden for security reasons).
Trunk Host: SIP server or host address provided by the carrier.
Status:
Pending: Trunk is requested but not yet approved.
Active: Trunk is approved and ready for use.
Action:
Delete: Remove the trunk request (only applicable if allowed).

Trunk Status Flow
Requested – Customer submits a trunk request.
Pending – Sohub/provider reviews the request.
Active – Trunk is approved and available for routing calls.
Only Active trunks can be used in:
Inbound Routes
Outbound Routes
Call Flow configurations

Important Notes
Pending trunks cannot place or receive calls.
SIP credentials are shown partially or hidden for security.
Trunk details should not be shared publicly.
If approval is delayed, contact Sohub support.

Best Practices
Verify provider details before requesting a trunk.
Use approved trunks only when configuring routes.
Regularly review trunk status to avoid call failures.
